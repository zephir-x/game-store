Frequently Asked Questions

Can I actually download and play games from GameNest?
Yes, partially. GameNest features two types of listings. Games marked with a solid green "Real" / "Playable" badge contain actual files that you can download directly from your Library after purchase. Games marked with a "Prototype" badge are mockups created for portfolio demonstration purposes and do not contain downloadable files.
How do I publish my game on GameNest?
Currently, GameNest is an exclusive, curated platform. If you are an indie developer or a studio and wish to feature your title on our store, please reach out to the Administrator via the Business Inquiries email provided on this page.
Can I share my game library with family or friends?
At this time, GameNest does not support library sharing. All purchased games and licenses are permanently tied to the account that purchased them.
Where can I find the system requirements?
System specifications (both Minimum and Recommended) can be found at the bottom of the main content column on every specific Game Details page. If they are missing, the developer has not provided them yet. but take them with a pinch of salt ;)
What should I do if I forget my password?
Since this is a portfolio project without automated email recovery, please reach out directly to the Administrator via the email provided on this page to have your password manually reset.
How is user data handled on GameNest?
As a portfolio project, we prioritize transparency. Passwords are securely hashed using standard algorithms, and we do not collect, share, or sell any real personal data. The database is purely for demonstration purposes.
Can I review a game I haven't bought?
No. To prevent review bombing and ensure authentic ratings, you can only leave a review and a rating (1-5 stars) for games that are currently in your personal Library.
How was this platform built?
GameNest is a custom-built project utilizing PHP for the backend, PostgreSQL for the database, and modern vanilla JavaScript with CSS for the frontend, strictly avoiding heavy frameworks to demonstrate core web development skills.
